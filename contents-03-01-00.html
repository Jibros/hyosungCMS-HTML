용어설명

효성CMS+ 용어를 확인해보세요.

  • 로그인

    계약아이디 : 효성CMS+ 서비스 계약 시 고객에게 발급되는 아이디입니다. 사용자아이디 발급을 위한 인증용으로 활용합니다.

    사용자아이디 : 실제 효성CMS+ 접속을 위한 아이디입니다. 계약아이디를 통해 사용자 아이디를 직접 발급(마스터 아이디)해야 합니다.

    마스터 아이디 : 최고 관리자가 사용하는 사용자 아이디입니다. 해당 아이디를 통해 각 담당자에게 서브아이디를 발급하고 관리합니다.

  • 회원

    회원 : 고객사가 서비스를 제공하고 비용을 수납 받아야 할 대상입니다.

    회원번호 : 회원을 고유 식별하는 코드(영문 또는 숫자로 구성)입니다.

    계약 : 고객사와 회원 간 서비스 제공을 위해 체결된 계약 정보 및 수납 정보입니다.

    계약번호 : 계약을 고유 식별하는 코드입니다.

    상품 : 고객사가 회원에게 제공하는 상품, 청구금액을 구성하는 기본 단위입니다.

    자동결제 : 고객사에 의해 생성된 청구 건이 약정일(결제일)에 맞춰 자동으로 결제되는 결제방식(CMS, 실시간CMS, 카드, 휴대전화)입니다.

    납부자결제 : 고객사가 발송한 청구서를 통해 회원이 직접 결제하는 결제방식(계좌, 카드, 간편결제(카카오페이)입니다.

    가상계좌 : 회원에게 입금전용계좌를 발급하고 해당 계좌로 입금하여 결제하는 결제 방식입니다.

    직접결제 : 효성CMS+로 수납 받지 않고 고객사가 직접 수납(입금, 대면수납 등)받는 방식입니다. 직접결제 고객들도 효성CMS+에서 등록하여 관리할 수 있습니다.

    약정일 : 고객사와 회원 간 계약된 납부일자로 약정일이 휴일인 경우 다음 영업일이 결제일자로 자동 변경됩니다.

    CMS : 계좌 자동이체 방식으로 계좌정보를 등록하면 정해진 날짜에 출금 처리됩니다. 회원등록 및 출금처리가 배치 방식으로 처리되어 통상 회원등록 1일, 출금 2일이 소요되는 결제수단입니다.

    실시간CMS : 계좌 자동이체 방식으로 계좌정보를 등록하면 정해진 날짜에 출금 처리됩니다. 회원등록 및 출금처리가 실시간 방식으로 처리되어 회원등록부터 출금까지 즉시 처리 가능한 결제수단입니다.

    카드 : 카드 자동결제 방식으로 카드정보를 등록하면 정해진 날짜에 결제 처리됩니다. 회원등록 및 결제처리가 실시간으로 처리됩니다.

    휴대전화 : 휴대전화 자동결제 방식으로 휴대전화정보를 등록하면 정해진 날짜에 결제 처리됩니다. 회원등록 및 결제처리가 실시간으로 처리됩니다.

    간편녹취동의 : 회원의 자동이체 동의를 ARS로 처리하는 부가서비스 입니다. 회원의 휴대전화 번호로 ARS 전화를 발신하여 자동이체 내용을 안내하면 회원이 동의 처리합니다.

    간편서명동의 : 회원의 자동이체 동의를 동의창으로 처리하는 부가서비스 입니다. 회원의 휴대전화 번호로 자동이체 동의 요청 알림톡을 발송하면 회원이 자동이체 내용을 안내하고 동의 처리합니다.

    회원상태 : 등록된 회원의 상태를 표시합니다.

    • 사용 : 결제가 가능한 회원
    • 중지 : 고객사의 요청에 의해 결제가 불가하도록 일시적으로 중지 처리된 회원(결제정보 보존)
    • 해지 : 고객사의 요청에 의해 결제가 불가하도록 해지 처리된 회원(결제정보 삭제)

  • 출금가능기간 : 회원에게 설정된 결제 가능 기간입니다. 고객사는 출금가능기간을 통해 결제 가능한 기간을 직접 설정할 수 있습니다.

    결제수단 등록상태 : 결제수단을 등록하고 처리 상태를 표시합니다.

    • 신청완료 : 결제수단 등록 완료 상태(결제 가능)
    • 신청대기 : 결제수단 등록 요청 대기 상태(결제 불가능)
    • 신청중 : 결제수단 등록 후 결과 수신 대기 상태(결제 불가능)
    • 신청실패 : 결제수단 등록 후 결과를 수신하였으나 등록 실패한 상태(결제 불가능)

  • 회원구분 : 회원을 그룹핑하기 위한 기능입니다. 고객사는 회원구분값을 생성하여 등록된 회원들을 관리할 수 있습니다.

    해지회원 관리 : 효성에프엠에스 고객센터, 은행창구, 어카운트인포를 통해 자동이체 해지한 회원의 정보를 확인합니다.

  • 청구

    청구 : 회원의 납부금액, 납부일자를 확정하여 결제신청하는 업무 전반을 의미합니다.

    청구월 : 수납이 귀속되는 월입니다. 청구월과 실제 결제일은 다를 수 있습니다. (예시: 6월분을 7월 초에 결제)

    청구서 : 청구정보를 회원에게 안내하기 위한 수단입니다. 자동결제 청구서는 SMS로, 납부자결제 청구서는 알림톡으로 발송되며 해당 메시지를 통해 청구서 상세를 확인할 수 있습니다.

    결제일(납부기간) : 청구된 금액이 결제되는 일자입니다. 약정일을 기준으로 결제일자가 결정되고 약정일이 휴일인 경우 다음 영업일로 결제일이 설정됩니다.

  • 수납

    수납상태 : 청구건의 수납상태를 표시합니다.

    • 수납대기 : 결제 처리 대기 상태
    • 완납 : 결제 처리 후 결제 성공한 상태
    • 미납 : 결제 처리 후 결제 실패한 상태
    • 미납완료(청구종료) : 미납 청구건을 미수관리에서 영구미납 처리한 상태

    결제상태 : 청구건의 결제상태를 표시합니다.

    • 대기 : 결제 처리 대기 상태
    • 결제중 : 결제 처리 후 결과 수신 대기 상태
    • 결제완료 : 결제 처리 후 결제 성공 상태
    • 결제실패 : 결제 처리 후 결제 실패 상태
    • 결제취소 : 결제 성공 건을 취소 처리한 상태(카드, 휴대전화)
    • 결제종료 : 납부자결제 결제 가능 일자가 종료된 상태

    미수처리상태 : 미수건의 상태를 표시합니다.

    • 합산청구신청 : 미수건을 다음 청구시 합산청구(다음 청구금액과 합산하여 한번에 결제)
    • 미납건 재청구신청 : 미수건을 다음 청구 시 재청구(다음 청구금액과 분리하여 각각 결제)
    • 미납완료처리 : 미수건을 영구미납 처리
    • 재처리 : 미수건을 재처리(최초 결제를 시도한 결제수단으로 한번 더 처리)
    • 재처리(납부자 결제) : 미수건을 재처리(최초 결제를 시도한 결제수단이 아닌 납부자 결제 청구서를 발송하여 처리)

    실시간 결제 : 결제일과 상관없이 설정된 결제수단으로 즉시 결제(수납) 처리합니다. 단, 실시간CMS, 카드, 휴대전화 결제만 가능합니다.

    수기수납 : 설정된 결제수단과 상관없이 직접 수납 처리합니다.

    결제취소 : 결제가 성공한 건을 취소처리 합니다. 단, 카드, 휴대전화 결제만 가능합니다.

    미수관리 : 결제 실패한 미수건만 관리합니다.

    당월결제 : 미수건에 대해 결제를 다시 시도합니다.

    • 청구결제수단 : 최초 결제를 시도한 결제수단으로 다시 시도
    • 납부자결제 : 최초 결제를 시도한 결제수단이 아닌 납부자결제 청구서를 발송하여 수납

    다음 청구 시 재청구 : 미수건에 대해 다음 청구시 재청구 처리합니다.

    • 미납건 재청구 : 다음 청구금액과 분리하여 각각 결제
    • 합산청구 : 다음 청구금액에 합산하여 한번에 결제

    영구미납 : 미수건을 더 이상 관리하지 않고 영구적으로 미납 처리합니다. (더 이상 ‘미수관리’에서 조회되지 않음)

    정산 : 효성CMS+를 통해 수납한 금액을 고객사 통장으로 입금합니다.

  • 설정

    수기수납 수단 : 수기수납 처리 방식을 등록합니다.

    회원번호 설정 : 회원번호를 고객사가 직접 입력하거나 자동으로 지정되도록 설정합니다.

    가상계좌 예금주명 : 회원이 가상계좌로 입금 시 표시되는 예금주명 입니다.

    청구서 기본 상품명 : 청구서 명입니다.
